# Security Policy

These scripts are defensive, read-only checks for systems you own or are authorized to audit.

## Supported Use

- Run on your own server or a client-approved server.
- Inspect the script before running it.
- Preserve suspicious files and logs before cleanup when compromise indicators are found.

## Not Supported

- Unauthorized scanning or access.
- Exploit development or weaponization.
- Credential theft, persistence, or bypass guidance.

## Report a False Positive

Open a GitHub issue with the scanner output after removing IP addresses, hostnames, usernames, tokens, and customer data.

For private repair help, use: https://ping7.cc/cve-repair
